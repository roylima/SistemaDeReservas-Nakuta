El UML (Lenguaje Unificado de Modelado) es un estándar gráfico utilizado para visualizar, especificar, construir y documentar los artefactos de un sistema de software. Aunque tradicionalmente asociado al paradigma orientado a objetos, en este contexto nos permite representar clases, objetos y sus interacciones para complementar el análisis estructurado, ofreciendo una visión más clara de la arquitectura estática del sistema ÑAK'UTAS.

El Diagrama Entidad-Relación (DER) es una herramienta conceptual utilizada en la fase de análisis para representar gráficamente la estructura lógica de la información. Captura las entidades principales del negocio (como Clientes, Servicios, Productos) y define las relaciones y cardinalidades entre ellas. Es el paso previo fundamental para el diseño de la base de datos, asegurando que la información esté organizada de manera coherente con las reglas de negocio de la peluquería.

El Modelo Relacional representa la implementación lógica del diagrama E-R. Aquí, las entidades se transforman en tablas (relaciones), definiendo atributos específicos, claves primarias (para identificación única) y claves foráneas (para vincular tablas). Este modelo es la base sobre la cual se construye la base de datos física del sistema.

El Diccionario de Datos es un repositorio centralizado que describe en detalle cada elemento de dato que fluye por el sistema. Proporciona definiciones precisas de los flujos de datos, almacenes y procesos, especificando nombres, alias, tipos de datos, longitudes y descripciones. Su propósito es asegurar una interpretación unificada de la información entre todos los analistas y desarrolladores del proyecto.
